# Исправление ошибки "role is required" при создании токена Hugging Face

## Проблема
При создании/редактировании токена появляется ошибка: `"role" is required`

## Решение

### Вариант 1: Создать новый токен (рекомендуется)

1. **Удалите старый токен** (если он есть) или создайте новый:
   - Перейдите на https://huggingface.co/settings/tokens
   - Нажмите "New token"

2. **При создании токена заполните все поля:**
   - **Name** (Имя токена): например, "Football News Bot"
   - **Type** (Тип): выберите "Read" или "Write" (для генерации изображений нужен "Write")
   - **Role** (Роль): **ОБЯЗАТЕЛЬНО выберите роль!**
     - Выберите любую доступную роль из списка
     - Обычно есть варианты типа "User", "Admin" и т.д.
     - Если роли не видны, попробуйте выбрать "Write" тип токена

3. **Включите права:**
   - В разделе **"User permissions"** → **"Inference"**
   - ✅ Отметьте **"Make calls to Inference Providers"**
   - ✅ Отметьте **"Make calls to your Inference Endpoints"** (опционально)

4. **Сохраните токен** и скопируйте его

5. **Обновите токен в config.py:**
   ```python
   HUGGINGFACE_API_KEY = "ваш_новый_токен"
   ```

### Вариант 2: Использовать существующий токен через API

Если не можете создать токен через интерфейс, можно попробовать использовать существующий токен, но он может не иметь нужных прав.

### Вариант 3: Временно отключить генерацию изображений

Если не получается настроить токен, можно временно отключить генерацию изображений:

В `config.py`:
```python
ENABLE_IMAGE_GENERATION = False
```

Бот будет работать и публиковать новости с прогнозами, но без изображений.

## Проверка работы

После обновления токена запустите бота:
```bash
python bot.py
```

Если токен настроен правильно, в логах вы увидите:
```
✅ Изображение сохранено: images/...
```

Если все еще ошибка 403, значит токен не имеет прав "Make calls to Inference Providers".

📘 Техническое задание

Проект: Платформа пополнений и выводов для казино с Telegram-ботом и админ-панелью
Цель: Автоматизировать пополнение и вывод средств пользователей через банки (C2C), с системой рефералов, историей операций и автообновлением балансов.

🔹 1. Общая архитектура

Проект состоит из трёх частей:

Telegram-бот (пользовательский интерфейс)

Основное взаимодействие пользователей: пополнение, вывод, история, рефералы, поддержка.

Работает на Python (библиотека telebot или aiogram).

Подключён к бекенду сайта через API.

Веб-сайт (клиентская часть)

Отображает те же функции, что и бот (для пользователей, предпочитающих браузер).

Страницы: пополнить, вывести, история, рефералы, инструкция, поддержка.

Может быть на Next.js, React или любой фреймворк (по выбору разработчика).

Админ-панель (веб-интерфейс)

Управление пользователями, транзакциями, балансами, статистикой, выплатами по рефералам.

Отдельный домен или поддомен (admin.site.com).

Защищён авторизацией.

🔹 2. Telegram-бот

Команда /start

Отправляет приветственное сообщение.

Показывает меню кнопок:

💰 Пополнить

💸 Вывести

📜 История

👥 Рефералы

🧾 Инструкция

🎧 Поддержка

2.1. Пополнение

Процесс:

Пользователь выбирает "Пополнить".

Бот предлагает выбрать казино (из списка, например: 1Win, Pin-Up, MostBet).

Затем бот просит ввести:

ID игрока (логин в казино)

Сумму пополнения

После этого:

Бот отправляет пользователю ссылку на оплату (QR или ссылка C2C).

Ссылка формируется автоматически по шаблону банка (Optima, Demir, MegaPay и т.д.).

Сумма и ID вставляются динамически, а контрольная сумма (хэш) генерируется по SHA-256 → последние 4 символа.

После оплаты:

Система парсит письмо из почты банка (или webhook), где приходит уведомление о поступлении.

Когда находится совпадение по сумме и ID, бот автоматически пополняет баланс пользователя.

Бот отправляет сообщение:

✅ Ваш баланс успешно пополнен!
Сумма: 5000 сом
Казино: 1Win

2.2. Вывод

Процесс:

Пользователь выбирает "Вывести".

Бот спрашивает:

Казино (из списка)

ID игрока

Сумму

Банк (куда отправить)

Номер телефона получателя

Код с сайта (для подтверждения)

После подтверждения:

Заявка отправляется в админ-панель.

Админ видит все заявки на вывод, подтверждает/отклоняет.

Пользователь получает уведомление о статусе:

💸 Ваша заявка на вывод отправлена администратору.
Ожидайте подтверждения.

2.3. История

Показывает последние операции (пополнения и выводы):

📜 История:
+5000 сом — Пополнение (1Win)
-2000 сом — Вывод (Pin-Up)

2.4. Реферальная система

Идея:

Каждый пользователь получает персональную ссылку для приглашения:

Ваша ссылка: t.me/YourBot?start=ref12345


Когда новый пользователь заходит по этой ссылке, он закрепляется как реферал.

За каждую активность рефералов (например, их пополнения) начисляется процент (например, 5%).

В конце месяца система автоматически:

Подсчитывает общий доход рефералов.

Начисляет бонус на баланс.

Обнуляет реферальную статистику до следующего месяца.

Пример:

👥 Реферальная программа:
Приглашено: 12 человек
Доход за месяц: 1250 сом
Следующая выплата: 1 ноября

2.5. Поддержка

Кнопка открывает Telegram-юзер админа или чата поддержки.

Можно добавить шаблонное сообщение:

💬 Напишите вашу проблему, и администратор свяжется с вами в ближайшее время.

🔹 3. Веб-сайт (для пользователей)

Разделы сайта (вкладки):

Раздел	Функционал
Пополнить	Аналогично боту: выбор казино, ID, сумма, генерация ссылки
Вывести	Форма вывода с кодом подтверждения
История	Таблица транзакций
Рефералы	Личный реферальный код, количество приглашённых, статистика
Инструкция	Текст с пошаговой инструкцией по оплате
Поддержка	Контакты, кнопка написать в Telegram
🔹 4. Админ-панель

Основные разделы:

📊 Главная панель — общая статистика:

Количество пользователей, общие пополнения, выводы, прибыль, активные рефералы.

💰 Пополнения — список всех пополнений (с фильтрацией и поиском).

💸 Выводы — список заявок на вывод, кнопки "подтвердить"/"отклонить".

👥 Пользователи — список всех пользователей с балансом и ID.

🤝 Рефералы — статистика, ручные корректировки и массовые начисления.

⚙️ Настройки — банки, API, SMTP почты (для парсинга писем), лимиты, проценты.

📩 Логи — отслеживание ошибок и событий.

🔹 5. Технические детали
Компонент	Технология
Бэкенд	Python (FastAPI / Flask)
Бот	Python (Telebot или Aiogram)
База данных	PostgreSQL / MySQL
Почтовый парсер	IMAP (парсинг писем банка)
Админ-панель	React / Next.js + Tailwind
Хостинг	VPS (Docker контейнеры)
Оплата	C2C через QR-ссылки (Optima, Demir, MegaPay и т.д.)
🔹 6. Безопасность

Авторизация JWT для API.

Проверка ID и сумм перед пополнением.

Код подтверждения при выводе.

Все действия логируются.

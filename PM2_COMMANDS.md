# Команды PM2 для просмотра информации о процессах

## Посмотреть детали процесса luxon-bot

```bash
# Показать полную информацию о процессе (включая путь к файлу)
pm2 show luxon-bot

# Или альтернативная команда
pm2 describe luxon-bot

# Показать информацию в JSON формате
pm2 jlist | jq '.[] | select(.name == "luxon-bot")'

# Показать только команду запуска
pm2 show luxon-bot | grep "script path"
```

## Посмотреть все процессы с деталями

```bash
# Показать все процессы
pm2 list

# Показать все процессы с детальной информацией
pm2 describe all
```

## Посмотреть логи процесса

```bash
# Показать логи процесса
pm2 logs luxon-bot

# Показать последние 100 строк логов
pm2 logs luxon-bot --lines 100

# Показать только ошибки
pm2 logs luxon-bot --err
```

## Посмотреть конфигурацию PM2

```bash
# Показать сохраненную конфигурацию
pm2 save

# Показать список процессов с их конфигурацией
pm2 prettylist
```

## Информация о конкретном процессе по ID

Если у вас есть ID процесса (например, 4 или 5), можно посмотреть:

```bash
# Показать информацию о процессе по ID
pm2 show 4

# Или
pm2 show 5
```

## Проверка конфигурационного файла

Также можно посмотреть файл конфигурации PM2:

```bash
# Посмотреть ecosystem.config.js
cat /var/www/luxon/ecosystem.config.js

# Или если файл в корне проекта
cat ecosystem.config.js
```

## Из конфигурации видно:

Из `ecosystem.config.js`:
- Процесс `luxon-telegram-bot` запускается из `/var/www/luxon/bot_simple`
- Файл: `bot.py`
- Команда: `python3 bot.py`

Если у вас процессы с именем `luxon-bot` (не `luxon-telegram-bot`), они могут быть запущены вручную или из другого конфига.
